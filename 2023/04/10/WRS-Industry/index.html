Posted on 2023-04-10 | In Web Recommender System

Insurance domain

• Many users navigate through the insurance website but still prefer to purchase over the phone. User actions and purchase data

• The insurance domain is data-sparse because of few different items and low purchase frequency. Graph based approach

• Customers have dynamic needs of insurance products as they are closely connected to life events. Session-based approach

The simple UB-KNN is better than the latent factor models.

The graph-based models generally perform well.

NGCF performs best, but Genetically Optimized Graph obtains competing results

Cross-sessions Model

Model: RNN with a single GRU-layer.

Input: A user’s multiple past sessions.

Output: What items the user will buy after the last time step.

Cross-sessions outperforms SKNN

Hybrid of Cross-sessions and Demographic Mode

Pleaz

Content-hosting websites traditionally want their users to have a personalized experience. Personalization increases user satisfaction, user satisfaction increases revenue

Planning the recommendation system:

Data -> Method -> Implementation

Data

  • Data/signal collection

  • Data preprocessing

​ - excluding company data

​ - excluding test/noise data points which exist because of technical/sync errors

  • Video localizations pairing with originals
  • Connecting user behavior data to user accounts
  • Distinguishing between company VS personal logins
Recommendation methods tested:

  • Random recommendations

  • Top-K (most popular videos)

  • Collaborative filtering (surprise toolbox for Python)

  • Content based (based on video titles + descriptions)

Integration

Core concept
  1. WordPress site connects to an external service (API)
  2. API responds with a recommendation list
  3. WordPress uses it to display recommendations
Challenge(s)?

Python code running as an API

Keeping an up-to-date version of the dataset

Already-seen videos - multiple refreshes within the same day.

It needs to be available all the time - no downtime for the server to retrain.

Also, don’t crash the main site at any point!

Why not even larger effect on the KPIs?

To align with our editorial values after scale up, we

• Limited the types of news content that could be recommended,

• Only recommended to readers who accepted profiling

• Constrained model training to avoid feedback loops that could create filter bubbles

• Among other things
