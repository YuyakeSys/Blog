VBO & EBO & VAO

Vertex buffer objects

- Describe what they are

Buffer object that uses the target 

GL_ARRAY_BUFFER
  • Contain vertex data, one or more attributes pervertex
  • Buffer size: Number of vertices * Size of all attributes
  • Each attribute stored separately
- List different vertex attributes
  • Position: 3D space
  • Color
  • Normal
  • Tangent
  • Texture coords

Element Buffer Object

- Describe what they are
  • Buffer object that uses the target
GL_ELEMENT_ARRAY_BUFFER

  • Contains a list of numbers (indices) of the vertices to be used

  • Buffer size: Number of primitives Vertex per primitive Size of integer type

- What is the benefit of using them?

Allows to reuses vertices in primitives

Vertex Array Object

• Describe what they are
  • Subtype of OpenGL objects (but not a buffer object)
  • Contains the information required to draw geometry
  • Does not store the data itself, just references
  • Bind it before calling glDrawArrays or glDrawElements functions
• Relation to VBO and EBO

Keeps pointers to all the vertex attributes stored in vertex buffer objects (VBO)

it specifies an element buffer object (EBO)

What is the winding order

• Triangle facing determined by winding order

The winding order has attribute GLENUM mode with clockwise and counter clockwise for drawing the triangle

Culling: Discard back and/or front faces

Vertex specification

Vertex processing (programmable)

• Process each vertex individually, in parallel

• Behavior is defined by shaders

Transform vertex attributes -> Transform postion

Vertex post-processing

• Transform feedback (optional)

• Can be used to store transformed vertices back into buffer objects

Primitive assembly

Split the input in a sequence of individual basic primitives

Rasterization

• Transform each primitive to the screen

• Breaks it into fragments

• Interpolates vertex data to fragments

Fragment processing (Programmable)

• Process each fragment individually, in parallel

• the behavior is defined by fragment shaders

Per-sample operations

Writes the pixel color, blending the fragment color with the previous content in the draw buffer

What is a shader and a shader program

Shader: Computer program that runs on the GPU

Single instruction, multiple data (SIMD)

Shader program: Set of shaders required for the entire pipeline

List all the different shader types

  • input and outputs of each one
  • Pipeline stage and order in which they are executed
  • Compute shaders X

Vertex shader

One input vertex -> at least position is provided as an attribute

One output vertex -> output position in clip coordinates to gl_Position / Can have any other output attributes

Geometry shader

Input： One primitive, a collection of vertex attributes from previous stage arrays

Output: zero or more output primitives, multiple vertices per invocation, not arrays

e.g One line primitive expands to 2 triangles

Fragment shader (in Fragment processing)

Input: attribute from last stage, usually vertex shader

Output: main output is a color

Most common format is 4 components RGBA within in (0.0 - 1.0)

If depth test enabled, additional depth output

Compute shader

Not using rasterization rendering pipeline
