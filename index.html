IEnumerable<string> FindFiles(string folderName)
{
    List<string> salesFiles = new List<string>();

    var foundFiles = Directory.EnumerateFiles(folderName, "*", SearchOption.AllDirectories);

    foreach (var file in foundFiles)
    {
        // The file name will contain the full path, so only check the end of it
        var extension = Path.GetExtension(file);
        if (extension == ".json")
        {
            salesFiles.Add(file);
        }
    }

    return salesFiles;
}

• Ray tracing vs rasterization

• Rasterization:
• For each object
• For each pixel (covered by the object)
• Test if is closer than the previously rendered (Depth buffer)
• Ray tracing:
• For each pixel
• For each object
• Test if it is the closest in that direction

• With one ray per pixel and no lighting, results are equivalent
• Rasterization:
• Lighting: Uses equations to estimate the amount of light received
• Shadows: Need to fake them in other ways (i.e. shadow maps)
• Ray tracing:
• Lighting: Cast rays to the light and other objects and accumulate light
• Shadows: Part of the algorithm, if objects block the rays to the light
• Can be recursive for more accurate lighting

• Ray tracing vs ray marching

Both methods start with rays from each pixel
• Ray marching:
• Many steps for one ray
• Test for closest distance
• Finish when it finds a hit
• Ray tracing:
• One single step per ray
• Test for collision
• Ray can bounce and create more rays

• What is a BVH and why it is useful

Good quality ray tracing requires many rays
• In real-time, we need to optimize the intersection with the scene

acceleration structure

Bounding Volume Hierarchy

• Organize the scene using a tree structure
• Each object is a leaf node
• Objects need to define a simple bounding volume:

• Benefits and limitations of ray tracing

Benefits: Better reflect dynamic objects, not limited to pixels.

Better shadows, not limited by shadow map resolution. allow transparent and tinted shadow

Better transparency, doesn’t require sorting

Refraction

Limitations:

Needs to test for colisions with every object
Complexity can grow exponentially
For good quality, many rays are needed
Can create noise when using path tracing

Gaussian blur

Blur: Smooth the color of a pixel based on the color of the surrounding pixels

  • Sample neighbor pixels and add the color multiplied by a weight
  • Total sum of weights need to be 1
  • The set of neighbors and their weights is named kernel
  • The weight is usually proportional to the distance to the pixel

Gaussian blur: Weights based on a gaussian distribution

The distance of the blur depends on the size of the kernel (linear interpolation)

What is HDR and tone mapping in games

HDR: High Dynamic Range

Dynamic range:

  • Ratio between the largest and small value that can be represented
  • Depends on many things (how we store color, display capabilities, etc)

Keep detail in both bright and dark areas in complex scenes

Rendering limited by precision of framebuffer:

RGBA8 -> 256 shades of light

Instead, use floating point framebuffers during rendering
• 4 times more memory
• Not clamped by the max value of 1
• Map to a good range when displaying the final image

Tone mapping

Map the high dynamic range image to a normalized (0-1) range

  • High exposure: dark -> brighter
  • bright values too bright -> can’t see detail
  • Low exposure: Dark values too dark, bright less bright

Many games simulate this with a dynamic exposure:

  • Compute the luminance of every pixel
  • Scale down, averaging neighboring pixels
  • Repeat until we have a single exposure value
  • Blend with exposure value from previous frame
  • Use that exposure value for tone-mapping

What is anti-aliasing and mention some techniques

• Aliasing: Visual stair-stepping of edges that occurs in an image
when the resolution is too low

• Anti-aliasing: Techniques targeted to reduce aliasing
Some of them are applied as post-processing

Super sampling

Render at a higher resolution (x2, x4,…) and downscale to target resolution. Not good for performance

Multi sampling -MSAA

Tries to preserve the benefits of super sampling
• While using less memory

Fragment per pixel, depth test per-sample

and average the result

Morphological - FXAA, MLAA, SMAA

Try to detect high contrast edges in image and blur

Pros: easy to implement in pipeline, works over the final image / can correct aliasing not caused by geometry

Cons: smooth anything

Temporal - TAA

Same concept as MSAA, sample in small offsets within pixel, but one each frame and blend previous frame

float rayMarch(vec3 origin, vec3 dir)
{
	float distance = 0.0f;
	for(int i = 0; i < maxSteps; ++i)
	{
		vec3 p = origin + dir * distance;
		float d = GetDistance(p);
		distance += d;
		if (distance > maxDistance)
			discard;
		if (d < surfaceDistance)
			break;
	}
return distance;
}

Lecture1

Feature of a Recommender System

  • Personalised Recommendations → different recommendations for every user;

  • Maintain a user model or user profile, describes the users’s preferences;

  • Implicit or explicit behavioural signals → ethical concerns?

  • Account for the context and additional information;

  • Types of Recommender Systems: Collaborative Filtering and Content-Based

Problem Setting

User-Item Matrix

Goal: predict ratings for unseen items

Properties of the Ratings Matrix

Sparse matrix: a small subset of entries are specified

  • Ratings can be:
    • Continuous: level of like or dislike (rare);
    • Interval based ratings: 5 or 7-point scale, sometimes 10 points, e.g., 5-star ratings;
    • Ordinal ratings: categorical values, e.g., strongly disagree, disagree, neutral, agree, strongly agree;
    • Binary ratings: 2 options (forced choice), e.g., like or dislike;
    • Unary ratings: only positive preference (positive preference utility matrix), e.g., purchase

Implicit / Explicit Feedback

  • Explicit feedback: ratings explicitly provided by users, e.g., click on “like” button;
  • Implicit feedback, sometimes Positive-Unlabeled (PU):
    • Ratings are inferred from user actions;
    • Easier to obtain, transparent to users;
    • Almost no cost;
    • Harder to interpret and can be misleading: no information whether the user liked/disliked an item;

Long-tail Distribution

• Low frequency items:

    • Larger profit;
    • More difficult to recommend;

Collaborative Filtering Recommendation

• Idea: exploit past behaviour/opinions of an existing user community to predict items the current users will like in the future (collective intelligence);

• Input: ratings matrix (pure CF)

• Output:

A. Predicted rate for unobserved user-item;

B. List of top-k recommended items (or users);

User-based nearest neighbour recommendation:

• Similar users have similar ratings on the same item; •

Use ratings from neighbouring users;

Item-based nearest neighbour recommendation:

• Reverse the problem → similarity between items;

• Similar items are rated in a similar way by the same user;

• Use user’s own ratings on neighbouring items.

User-based Nearest Neighbour Recommendation

• Idea: given a target user:

  1. Identify users (peer users or nearest neighbours) that have similar preferences;
  2. For every unobserved item, make a prediction based on the preferences of peer users;

• Assumptions:

​ • Similar tastes in the past → similar tastes in the future;

​ • Preferences are stable and consistent over time.

Step 1: Similarity with Pearson Coefficient

• Simple approach: users with highest similarity score (Pearson coefficient);

• Problem: number of observed ratings can be very different depending on the item, e.g., popular items;

Step 2: Mean-centered Rating Prediction

Solution: center ratings by their row-mean

Use a weighted average to predict the rating:

Better Neighbourhood Selection?

• Include all users?

​ • Negative impact on accuracy and computing time;

• Minimum threshold on user similarity:

​ • Only positive correlation (negative correlations → inversion → not much predictive value);

​ • High threshold → small sets of neighbours → no predictions (reduced coverage);

• Top-k closest neighbours:

​ • High value for k → additional noise;

​ • Empirically: good values for k are in the range of [20, 50]

Item-based Nearest Neighbour Recommendation

similarity between items instead of similarity between users

Works better with offline pre-processing

Step 1: Cosine Similarity

Cosine similarity: empirically better than Pearson for item-based recommendation

Adjusted cosine similarity: mean centred ratings ;

Step 2: Rating Prediction

Consider the top-k similar items on a user basis;

set of K closest items (adjusted cosine similarity) to the target item that were rated by user ;
